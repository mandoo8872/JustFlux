## 2025-02-12 - Accessibility improvement on FileActions component
**Learning:** Found that custom file upload buttons (hidden `<input type="file">` wrapped in a `<label>`) are inherently ignored by keyboard navigation (Tab order) unless `tabIndex={0}` is added to the wrapping element. Additionally, they do not trigger click events naturally via Enter/Space; you need an explicit `onKeyDown` handler to call `click()` on the hidden input ref to achieve true keyboard accessibility.
**Action:** When creating custom file input buttons disguised as icons/labels, always add `tabIndex={0}`, `role="button"`, an `onKeyDown` handler for Enter/Space, and an `aria-label`.
